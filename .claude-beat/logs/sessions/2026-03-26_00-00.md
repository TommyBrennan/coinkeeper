## Decisions
- Used httpOnly cookies for sessions (not JWT) — more secure, server-validated

- `requireUser()` uses Next.js `redirect()` — works in both server components and route handlers
- Middleware checks cookie existence only (fast edge check); session validation happens server-side
- Registration creates user + credential + session atomically
