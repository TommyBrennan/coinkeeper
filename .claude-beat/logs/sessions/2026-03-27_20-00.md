## Docker Investigation
Tested 5 approaches to get Docker working:
1. `unshare --map-root-user` + dockerd (works for daemon, fails on image extraction — single UID can't handle multi-UID images)
2. VFS storage driver (same lchown failure)
3. `--map-users=auto` (needs setuid-root newuidmap, unavailable)
4. Real newuidmap binary (setuid-claude-beat, not root — kernel rejects)
5. dockerd-rootless.sh (needs ip command + proper UID mapping)
**Conclusion**: Deploy requires Docker socket access from host or DOCKER_HOST pointing to socket proxy.
